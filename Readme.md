# Arduino stepper potentiometer guide
        Этот скетч позволяет регулировать сторону вращения шагового двигателя 28BYJ-48 
    при помощи потенциометра. 
        При преодолении порогового значения мотор начинает двигаться в другую сторону.

# Принцип работы

Скетч непрерывно читает значение с потенциометра (0-1023) и сравнивает его с пороговым значением (THRESHOLD_POTENTIOMETR_VALUE).

- **Если значение > порога** → мотор делает 8 микрошагов (1 полный шаг) **по часовой стрелке**
- **Если значение < порога** → мотор делает 8 микрошагов **против часовой стрелки**

Таким образом, вращая ручку потенциометра, вы заставляете мотор повторять движение в соответствующую сторону.

# Схема подключения

## Подключение драйвера
Вход драйвера|Arduino Цифровой пин|
-------------|--------------------|
IN1          | PIN_8              |
IN2          | PIN_9              |
IN3          | PIN_10             |
IN4          | PIN_11             |

"+" Драйвера -> 5V Arduino
"-" Драйвера -> GND Arduino 
*!При подключении двигателей большего напряжения на + и - драйвера подаётся внешнее питание!*

Мотор через коннектор подключается к штекеру драйвера.

## Подключение потенциометра 

Левая ножка   -> 5V Arduino
Средняя ножка -> A0 Аналоговый пин Ардуино (пин можно изменить в скетче)
Правая ножка  -> GND Arduino

*!Рекомендуется производить подключение через макетную плату для удобства монтажа!*

# Переменные и константы проекта

## Конфигурационные константы (DEFINE)

### `SENSOR_PIN`
- **Тип:** `#define` (числовая константа)
- **Значение:** `A0`
- **Описание:** Определяет аналоговый пин Arduino, к которому подключен средний вывод потенциометра.
- **Диапазон:** A0-A5 (на Arduino Uno)

### `PIN_1`, `PIN_2`, `PIN_3`, `PIN_4`
- **Тип:** `#define` (числовые константы)
- **Значения:** `8`, `9`, `10`, `11`
- **Описание:** Цифровые пины Arduino, подключенные к входам драйвера ULN2003 (IN1, IN2, IN3, IN4 соответственно).
- **Диапазон:** 0-13 (цифровые пины)

### `DELAY_TIME`
- **Тип:** `#define` (числовая константа)
- **Значение:** `2`
- **Описание:** Время задержки между микрошагами в миллисекундах. Определяет скорость вращения мотора.
- **Рекомендации:**
  - `1-2 мс` - быстрое вращение (возможен пропуск шагов)
  - `3-5 мс` - оптимальная скорость для 28BYJ-48

### `THRESHOLD_POTENTIOMETR_VALUE`
- **Тип:** `#define` (числовая константа)
- **Значение:** `600`
- **Описание:** Пороговое значение считываемое с потенциометра. При превышении этого значения мотор вращается в одну сторону, при снижении - в другую.
- **Диапазон:** 0-1023
- **Рекомендация:** Использовать значение `1023/2 ± 100` (середина диапазона)

## Глобальные переменные

### `sensorValue`
- **Тип:** `int` (целое число)
- **Начальное значение:** `0`
- **Описание:** Хранит текущее значение напряжения, считанное с аналогового пина `SENSOR_PIN` (потенциометра). Обновляется в каждом цикле `loop()`.
- **Диапазон:** 0-1023
  - `0` - минимальное напряжение (0V)
  - `1023` - максимальное напряжение (5V)

## Таблицы и массивы

### `seq[8][4]`
- **Тип:** двумерный массив `int`
- **Размер:** 8 строк × 4 столбца
- **Описание:** Таблица последовательности состояний для полушагового режима управления шаговым двигателем 28BYJ-48. Каждая строка представляет собой один микрошаг и содержит значения для четырех пинов (PIN_1, PIN_2, PIN_3, PIN_4).
- **Значения в таблице:**
  - `1` - HIGH (включено)
  - `0` - LOW (выключено)
- **Строки таблицы (микрошаги):**
  - Строка 0: `{1, 0, 0, 0}` - активен только PIN_1
  - Строка 1: `{1, 1, 0, 0}` - активны PIN_1 и PIN_2
  - Строка 2: `{0, 1, 0, 0}` - активен только PIN_2
  - Строка 3: `{0, 1, 1, 0}` - активны PIN_2 и PIN_3
  - Строка 4: `{0, 0, 1, 0}` - активен только PIN_3
  - Строка 5: `{0, 0, 1, 1}` - активны PIN_3 и PIN_4
  - Строка 6: `{0, 0, 0, 1}` - активен только PIN_4
  - Строка 7: `{1, 0, 0, 1}` - активны PIN_1 и PIN_4
- **Примечание:** 8 микрошагов (полная таблица) составляют 1 полный шаг двигателя. Для полного оборота двигателя 28BYJ-48 требуется 4096 микрошагов (512 полных шагов).

## Условные константы отладки

### `DEBUG`
- **Тип:** макрос (`#define`)
- **Описание:** При раскомментировании строки `//#define DEBUG` включает режим отладки, выводя значения `sensorValue` в монитор порта.
- **Использование:** 
  - `//#define DEBUG` - отладка отключена (режим по умолчанию)
  - `#define DEBUG` - отладка включена


# Возможные проблемы

| Проблема | Решение |
|----------|---------|
| Мотор не крутится | Проверьте подключение пинов 8,9,10,11 и питание драйвера |
| Мотор пропускает шаги | Увеличьте `DELAY_TIME`, проверьте питание |
| Значения в мониторе порта не меняются | Проверьте подключение потенциометра (средний пин обязательно на A0) |
| avrdude: stk500_recv(): programmer is not responding| Проверьте видимость COM порта, переподключите Arduino, перезапустите IDE|
